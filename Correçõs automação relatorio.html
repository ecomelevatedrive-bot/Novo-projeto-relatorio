Bugs Corrigidos
BUG MONDAY_BOARD_ID incorreto config.py · testar_monday.py
O ID do board estava errado em ambos os arquivos, fazendo com que os relatórios fossem enviados para um board inexistente.
  • Antes: 18414132966
  • Depois: 18410901694 (Ezequiel - Execução)
BUG CRÍTICO UnicodeEncodeError — terminal fechava imediatamente RODAR_RELATORIO.bat
O bat chamava Python sem flag UTF-8. Caracteres como → no código causavam crash imediato e o terminal fechava sem mostrar erro.
  • Corrigido: adicionado -X utf8 em todas as chamadas Python
  • Também adicionado chcp 65001 no bat para garantir encoding correto
BUG Comando tee não existe no Windows RODAR_RELATORIO.bat
O bat usava | tee para salvar log e exibir na tela ao mesmo tempo. Esse comando não existe no Windows nativamente — causava fechamento silencioso.
  • Corrigido: substituído por > log.txt seguido de type log.txt
BUG Tarefa agendada falhava com exit code 1 tarefa_agendada.xml · RODAR_RELATORIO.bat
A execução automática de 22/05/2026 às 00:00 falhou silenciosamente (exit code 1) por causa do UnicodeEncodeError. Sem log, era impossível diagnosticar.
  • Corrigido: UTF-8 forçado resolve o crash
  • Log automático criado em logs/relatorio_AAAA-MM-DD_HHMM.log
BUG Monday.com retornava 429 Too Many Requests monday_integration.py
Sem tratamento de rate limit, a API do Monday rejeitava requisições durante execuções longas (muitos clientes) e o relatório não era enviado.
  • Corrigido: retry automático com espera progressiva (60s / 120s / 180s)
  • Até 3 tentativas antes de falhar definitivamente
BUG Script wise — rate limit sem retry (COMPLEXITY_BUDGET_EXHAUSTED) monday-tarefas-automaticas-board2.js
O delay de 200ms entre requisições era muito curto para criar 72+ itens no Monday. Vários clientes eram pulados (FORELI, Byelpe, ThaBeauty, Javali, etc.).
  • Identificado na execução de teste — clientes afetados listados no log
  • Correção pendente: aumentar delay e adicionar retry com retry_in_seconds
BUG Script de wake/sleep — PC não acordava TESTAR_WAKE_SLEEP.bat
O script original de teste tinha 4 problemas que impediam o PC de acordar:
  • schtasks com ^ quebrando linha falhava silenciosamente
  • WakeToRun via PowerShell não era salvo corretamente
  • Cálculo de hora com %time% quebrava quando hora < 10 (espaço extra)
  • errorlevel checado após PowerShell, não após schtasks
Corrigido: script reescrito usando XML com WakeToRun=true embutido.
